# 03 · 排期（动态排期 · 核心）

## TC-S-01 初次创建本自动排期
- **类型**: 正向
- **前置**: 创建一本有 5 画页共 12h 的本，deadline 14 天后
- **预期**: 系统自动给出每日排期建议；总时长覆盖全部画页；临近 deadline 的日子稍密一点（前倾）

## TC-S-02 均匀分摊验证
- **类型**: 正向
- **前置**: 12h 总量 / 14 天 / 每日 8h
- **预期**: 平均每天 1-2h 左右，不出现单日 >8h 或某些天全空

## TC-S-03 多本并行
- **类型**: 正向
- **前置**: 同一天内有 2 本有排期
- **预期**: 两本的画页混在同一天，总工时 ≤ 该日可用工时

## TC-S-04 手动拖拽覆盖
- **类型**: 正向
- **前置**: 有排期
- **步骤**: 把某画页从 D 日拖到 D+2 日
- **预期**: 拖后的项打上"已调整"标，后续重算不再自动移动它

## TC-S-05 请假日不排单
- **类型**: 正向
- **前置**: 排期中间某天请假
- **步骤**: 在日历上点击该日的"请假"
- **预期**: 该日原有排期自动顺延到下一个可用日

## TC-S-06 请假取消后恢复
- **类型**: 正向
- **前置**: 某日已请假
- **步骤**: 点击"取消请假"
- **预期**: 后续排期按新可用日重算（之前被顶走的项不一定回到原位，只要总量符合即可）

## TC-S-07 每日晚上重算（模拟）
- **类型**: 正向
- **前置**: 今日排期 3h，实际只完成 2h
- **步骤**: 晚上 22:00 提醒弹窗 → 填写已完成 2h
- **预期**: 明日起的排期把 1h 欠量重新分配到剩余日子

## TC-S-08 每日手动修改工时
- **类型**: 正向
- **前置**: 无
- **步骤**: 在日历把明天的工时改为 4h（平时 8h）
- **预期**: 明天的可容纳量减半，其余日子吸收溢出

## TC-S-09 deadline 过了仍未完成
- **类型**: 边界
- **前置**: 某本 deadline 昨天，今天未完成
- **预期**: 本状态=延期；排期不再自动铺新日子；提示建议延长 deadline 或标记完成

## TC-S-10 剩余天数 0 / deadline = 今天
- **类型**: 边界
- **前置**: 新建一本 deadline 设为今天
- **预期**: 所有画页堆在今天；若超过今日工时会在今天总量里显示溢出警示

## TC-S-11 排期算法公平性
- **类型**: 正向
- **前置**: 同一天 2 本截止日相近
- **预期**: 排期偏向先做交付日近的本（前倾权重）

## TC-S-12 封页优先
- **类型**: 正向
- **前置**: 一本含封页 + 其他画页
- **预期**: 封页排在最早的可用日

## TC-S-13 画页时长为 0.25 的最小步长
- **类型**: 边界
- **前置**: 添加一个 0.25h 的画页
- **预期**: 能被正确纳入排期（不被忽略）

## TC-S-14 超大本
- **类型**: 兼容
- **前置**: 一本含 50+ 画页
- **预期**: 性能可接受（排期生成 < 1s），UI 能滚动浏览
