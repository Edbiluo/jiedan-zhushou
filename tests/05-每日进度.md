# 05 · 每日进度 & 提醒

## TC-D-01 22:00 桌面通知触发
- **类型**: 正向
- **前置**: 开启桌面通知；今日未填报
- **预期**: 到 22:00 弹系统通知"今天画得怎么样？来填一下今日完成吧～"

## TC-D-02 已填报则不通知
- **类型**: 正向
- **前置**: 今日已填报
- **预期**: 22:00 不弹通知（避免打扰）

## TC-D-03 打开应用时的兜底弹窗
- **类型**: 正向
- **前置**: 昨天没填报，今早首次打开应用
- **预期**: 应用启动后立刻弹出填报弹窗（按昨天日期？或今天日期，见 TC-D-04）

## TC-D-04 兜底弹窗的日期归属
- **类型**: 边界 / 需求确认
- **说明**: 若 22:00 未填，第二天早上打开应用应该补填昨天还是跳过？
- **建议预期**: 仅针对"今天"未填报做兜底，历史日期不回看（避免越积越多）

## TC-D-05 填报后触发重算
- **类型**: 正向
- **前置**: 今日排期 3h，实际填 1h 且未完成
- **步骤**: 填报 notes + 隐式将未完成 2h 结转
- **预期**: 后续排期把剩余量分摊到明天起的可用日

## TC-D-06 填报后撤回
- **类型**: 正向 / 需求确认
- **说明**: 用户手滑填错，能否重新填报？
- **建议预期**: 能；覆盖当天 day_log 记录

## TC-D-07 关闭提醒时间段
- **类型**: 正向
- **前置**: 设置里把"启用桌面通知"关掉
- **预期**: 22:00 不再弹通知；兜底弹窗保留

## TC-D-08 修改提醒时间
- **类型**: 正向
- **前置**: 改为 21:30
- **预期**: 次日 21:30 触发通知

## TC-D-09 跨零点填报
- **类型**: 边界
- **前置**: 凌晨 00:30 打开应用
- **预期**: 兜底弹窗按"昨天"日期提示（避免凌晨画完没填还算"今天"）

## TC-D-10 没有安排的日子
- **类型**: 边界
- **前置**: 今天本来没任务 / 请假日
- **预期**: 不弹提醒或弹一个友好"今天休息～"气泡（二选一）

## TC-D-11 多次打开应用不重复弹
- **类型**: 正向
- **前置**: 同一天反复打开
- **预期**: 兜底弹窗只弹一次（填报后不再弹）

## TC-D-12 通知权限被系统拒绝
- **类型**: 异常
- **预期**: 降级为兜底弹窗，不崩溃
