## ⚠️ 已知问题 / 需要你留意

1. **首次 `npm install` 可能很慢**（全量下 Electron + sharp + better-sqlite3 native binaries），请有耐心
2. **`better-sqlite3` 在 Windows 首次打包时需要 electron-rebuild** —— `electron-builder` 会自动处理，但第一次构建可能额外等 2-3 分钟
3. **手写体字体**：`frontend/index.html` 引用了 CDN 版本的霞鹜文楷。如果你女朋友电脑没网 → 需要把字体文件下载到 `frontend/public/fonts/` 并改成本地路径（我没下字体文件，因为我没法从网上拉资源）
4. **测试用例 TC-D-04**（晚间通知若凌晨触发的日期归属）和 **TC-D-06**（填报后能否撤回）在用例里标了"需求确认"—— 当前代码按默认行为处理，如果你有倾向可以告诉我调整
5. **无真实数据**：应用首次启动是空数据库，她需要自己上传画页 / 建本。我没塞 demo 数据，保持纯净

## 🧪 自检提示

启动后推荐的"走通一遍"顺序：
1. **设置页** → 检查预置款式/尺寸是否存在
2. **画页库** → 上传 2-3 张图（各款式都来一张）
3. **本管理** → 新建一本，勾选几张画页，deadline 设 7 天后
4. **日历看板** → 看排期是否自动生成
5. **本详情** → 试试 ← → 拖动某个画页换日，验证"已调整"标
6. **随手勾一个画页完成** → 看日历联动
7. **标记整本完成** → 应该会看到撒花动效 🎉
8. **稿费统计** → 有一笔已完成 → 能看到数据
9. **设置 → 导出备份** → 看看能不能生成 zip
